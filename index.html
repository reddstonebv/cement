What are you building?

Everything else follows from this one.

1

Create the Local site

Local → Create a new site → PHP 8.1+, latest WordPress. Note the site domain and path — all cloning happens inside <site>/app/public/wp-content/.

Use plain http:// locally — Local's SSL cert is self-signed and only the mobile gyro features need HTTPS.
2

Clone the core + shell

cd wp-content/plugins
git clone git@github.com:reddstonebv/cement-wp-plugin.git wp-cement-plugin

cd ../themes
git clone git@github.com:reddstonebv/cement-wp-theme.git wp-cement-theme

Folder names matter — child themes declare Template: wp-cement-theme.

3

Fork the child starter

Clone with shared history so starter updates can be merged in later:

cd wp-content/themes
git clone git@github.com:reddstonebv/cement-wp-theme-child.git wp-cement-theme-<project>
cd wp-cement-theme-<project>
git remote rename origin upstream
git remote add origin git@github.com:reddstonebv/cement-wp-theme-<project>.git
git push -u origin main

Then work through the rename checklist: theme name, text domain, handle prefixes, token prefix. Existing project? Clone it instead — e.g. cement-wp-theme-rsr.

4

Activate & configure

Plugins → activate Cement. Appearance → Themes → activate your child theme. Settings → Permalinks → Post name (required for the /cement/* routes).

Create the pages your templates expect (slugs matter — templates resolve content by slug), set the static front page, assign the primary menu. Per-project details live in the project README — see RSR's for a worked example with a full page table.

5

Tooling (for committing)

git clone git@github.com:reddstonebv/cement-tooling.git ~/Local\ Sites/cement-tooling
ln -s ~/Local\ Sites/cement-tooling /tmp/cement-tooling
npm install   # in each repo you commit from

Pre-commit hooks run prettier + phpcs --standard=PSR12. macOS clears /tmp on reboot — recreate the symlink when commits fail with "config not found".

1

Consume the tokens

The Cement plugin serves the shared token set at /cement/css/brand.css. Load it before your own styles:

<link rel="stylesheet" href="https://<cement-host>/cement/css/brand.css" />
<link rel="stylesheet" href="css/site.css" />

All tokens are --cmt-* custom properties. Override only what differs, under your own project prefix. Consuming sites show up automatically in the plugin's Connected Sites view.

2

Share the tooling

npm install --save-dev github:reddstonebv/cement-tooling

Extend the shared stylelint/eslint/prettier configs — see the cement-tooling README for the per-project stylelint token-prefix rule.

3

Repo & deploy

Name it cement-static-<project> under reddstonebv. GitHub Pages, Netlify, or the client's host — the token CDN link is the only runtime dependency on Cement.
